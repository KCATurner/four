four.infer

Estimate the number of periods in letter-efficient numbers given a target name length.

CLI Subcommand

four infer

estimate letter-efficient numbers

usage: four infer [-h] [-g] [-w] [-q] targets [targets ...]
targets

Target number length(s)

-h, --help

show this help message and exit

-g, --graph

graph the resulting prediction(s)

-w, --whole

use equation with only whole numbers: F(x, a) = T(x) / a instead of default power equation: F(x, a, b) = a * T(x) ** b

-q, --quick

approximate targets by largest period: T(x) = 373 * 1000 ** x instead of using entire target value: T(x) = int(x)

four.infer.LENGTH_PERIODS_TO_TARGET_PERIODS: Tuple[int, ...]

Tuple mapping the number of periods P in the length L of a number N’s name, to the number of periods in N for the first 32 most letter-efficient numbers with names of length L, where when . When P = 0, L = 1 since no number can have fewer than 1 period.

Example

>>> n = PNumber("1113323[373]{6813}")
>>> n.name_length == PNumber("[373]{2}")
True
>>> LENGTH_PERIODS_TO_TARGET_PERIODS[2] == n.num_periods
True

Iterate over X yielding the approximate value of each PNumber.

Each PNumber x in X is approximated by multiplying the value of the leading period by the power of 1000 equal to the total number of periods in the number x.

four.infer.iter_exact_targets(X: Iterable[int | str | Iterable[int | Tuple[int, int] | RPeriod]]) → Generator[int, None, None][source]

Iterate over X yielding the exact value of each PNumber.

While the exact value of a PNumber is calculated using string multiplication and concatenations for performance reasons, mathematically it is a nested summation:

int(‘’.join(str(value).zfill(3) * repeat for value, repeat in self))
